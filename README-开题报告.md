## 如何转换为Word格式

### 方法一：使用Pandoc（推荐）

1. 安装Pandoc：https://pandoc.org/installing.html

2. 打开命令行，运行：
```bash
cd "C:\毕设项目"
pandoc "开题报告-技术文档智能问答系统-完整版.md" -o "开题报告.docx" --reference-doc=template.docx
```

3. 如果没有模板，直接运行：
```bash
pandoc "开题报告-技术文档智能问答系统-完整版.md" -o "开题报告.docx"
```

### 方法二：手动复制到Word

1. 打开Microsoft Word
2. 复制Markdown文件内容
3. 粘贴到Word中
4. 调整格式：
   - 标题样式（标题1、标题2、标题3）
   - 字体（正文：宋体/小四；标题：黑体）
   - 行距（1.5倍行距）
   - 页边距（上下2.54cm，左右3.17cm）

### 方法三：使用在线工具

访问：https://pandoc.org/try/
- 将Markdown内容粘贴
- 选择输出格式为DOCX
- 点击转换并下载

## 如何生成架构图

### 在线生成（最简单）

1. 访问 https://mermaid.live/
2. 打开 `系统架构图.md`
3. 复制任意一个图表代码（```` ```mermaid ... ```` 之间的内容）
4. 粘贴到左侧编辑器
5. 右侧自动生成预览
6. 点击 "Download PNG" 或 "Download SVG"
7. 保存图片到本地

### VS Code生成

1. 安装 VS Code
2. 安装插件 "Markdown Preview Mermaid Support"
3. 打开 `系统架构图.md`
4. 按 `Ctrl+Shift+V` 预览
5. 右键点击图片 → "Save as Image"

### 命令行生成

```bash
# 安装 Node.js
# 然后安装 mermaid-cli
npm install -g @mermaid-js/mermaid-cli

# 生成图片
cd "C:\毕设项目"
mmdc -i "系统架构图.md" -o "架构图.png"
```

## 格式调整建议

### Word样式设置

```yaml
正文样式:
  字体: 宋体
  字号: 小四 (12pt)
  行距: 1.5倍
  段前: 0行
  段后: 0行

标题1:
  字体: 黑体
  字号: 三号 (16pt)
  行距: 1.5倍
  段前: 1行
  段后: 0.5行

标题2:
  字体: 黑体
  字号: 四号 (14pt)
  行距: 1.5倍
  段前: 0.5行
  段后: 0.5行

标题3:
  字体: 黑体
  字号: 小四 (12pt)
  加粗: 是
  行距: 1.5倍
```

### 页面设置

```
纸张: A4
页边距:
  上: 2.54cm
  下: 2.54cm
  左: 3.17cm
  右: 3.17cm
页眉: 1.5cm
页脚: 1.75cm
```

### 图片格式

- 格式：PNG（推荐）或SVG
- 分辨率：至少150 DPI，推荐300 DPI
- 宽度：占页面宽度的80-100%
- 图片编号：图1、图2、图3...
- 图注格式：居中，小五号宋体，位于图片下方

## 常见问题

### Q1: Mermaid图显示异常怎么办？

**A**: 确保使用支持Mermaid的编辑器或在线工具。推荐使用 mermaid.live 在线生成。

### Q2: 转换为Word后格式乱了怎么办？

**A**:
1. 使用Pandoc转换时指定模板
2. 或手动调整Word样式（标题、正文、行距）
3. 确保原Markdown文件格式规范

### Q3: 图片插入Word后位置不对？

**A**:
1. 右键点击图片 → "插入和排列" → "嵌入型"
2. 设置图片居中对齐
3. 调整图片大小为合适宽度

### Q4: 参考文献格式需要调整吗？

**A**: 根据学校要求调整。当前格式为通用学术格式，如需特定格式（如GB/T 7714），请手动调整或使用工具。
